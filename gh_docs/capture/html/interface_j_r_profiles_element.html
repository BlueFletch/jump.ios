- (void) replaceFriendsArrayOnCaptureForDelegate: (id<JRCaptureObjectDelegate>)  delegate
context: (NSObject *)  context 

Use this method to replace the JRProfilesElement.friends array on Capture after adding, removing, or reordering elements. You should call this method immediately after you perform any of these actions. This method will replace the entire array on Capture, including all of its elements and their sub-arrays and sub-objects. When successful, the new array will be added to the JRProfilesElement.friends property, replacing the existing NSArray.

If the array is replaced successfully, the method replaceArrayDidSucceedForObject:newArray:named:context: (JRCaptureObjectDelegate-p) will be called on your delegate. This method will return a pointer to the new array, which is also the same pointer stored in the JRProfilesElement.friends property, and the name of the replaced array: "friends".

If unsuccessful, the method replaceArrayDidFailForObject:arrayNamed:withError:context: (JRCaptureObjectDelegate-p) will be called on your delegate.

Parameters:
delegate The JRCaptureObjectDelegate that implements the optional delegate methods replaceArrayDidSucceedForObject:newArray:named:context: (JRCaptureObjectDelegate-p) and replaceArrayDidFailForObject:arrayNamed:withError:context: (JRCaptureObjectDelegate-p).
context Any NSObject that you would like to send through the asynchronous network call back to your delegate, or nil. This object will be passed back to your JRCaptureObjectDelegate as is.Contexts are used across most of the asynchronous Capture methods to facilitate correlation of the response messages with the calling code. Use of the context is entirely optional and at your discretion.
Warning:
When successful, the new array will be added to the JRProfilesElement.friends property, replacing the existing NSArray. The new array will contain new, but equivalent JRFriendsElement objects. That is to say, the elements will be the same, but they will have new pointers. You should not hold onto any references to the JRProfilesElement.friends or JRFriendsElement objects when you are replacing this array on Capture, as the pointers will become invalid.
Note:
After the array have been replaced on Capture, you can now call JRFriendsElement.updateOnCaptureForDelegate:context:() on the array's elements. You can check the JRFriendsElement.canBeUpdatedOnCapture property to determine if an element can be updated or not. If the JRFriendsElement.canBeUpdatedOnCapture property is equal to NO you should replace the JRProfilesElement.friends array on Capture. Replacing the array will also update any local changes to the properties of a JRFriendsElement, including sub-arrays and sub-objects.
If you haven't added, removed, or reordered any of the elements of the JRProfilesElement.friends array, but you have locally updated the properties of a JRFriendsElement, you can just call JRFriendsElement.updateOnCaptureForDelegate:context:() to update the local changes on the Capture server. The JRFriendsElement.canBeUpdatedOnCapture property will let you know if you can do this.

- (BOOL) needsUpdate  

Use this method to determine if the object or element needs to be updated remotely. That is, if there are local changes to any of the object/elements's properties or sub-objects, then this object will need to be updated on Capture. You can update an object on Capture by using the method updateOnCaptureForDelegate:context:.

Returns:
YES if this object or any of it's sub-objects have any properties that have changed locally. This does not include properties that are arrays, if any, or the elements contained within the arrays. NO if no non-array properties or sub-objects have changed locally.
Note:
This method recursively checks all of the sub-objects of JRProfilesElement:
  • JRProfilesElement.profile
If any of these objects are new, or if they need to be updated, this method returns YES.
Warning:
This object, or one of its ancestors, is an element of a plural. If any elements of the plural have changed, (added or removed) the array must be replaced on Capture before the elements or their sub-objects can be updated. Please use the appropriate replace<ArrayName>ArrayOnCaptureForDelegate:context: method first. Even if needsUpdate (JRCaptureObject) returns YES, this object cannot be updated on Capture unless JRCaptureObject.canBeUpdatedOnCapture also returns YES.
This method recursively checks all of the sub-objects of JRProfilesElement but does not check any of the arrays of the JRProfilesElement or the arrays' elements:
  • JRProfilesElement.friends, JRFriendsElement
  • JRProfilesElement.following, JRFollowingElement
  • JRProfilesElement.followers, JRFollowersElement
If you have added or removed any elements from the arrays, you must call the following methods to update the array on Capture: replaceFriendsArrayOnCaptureForDelegate:context:, replaceFollowingArrayOnCaptureForDelegate:context:, replaceFollowersArrayOnCaptureForDelegate:context:
Otherwise, if the array elements' JRCaptureObject.canBeUpdatedOnCapture and needsUpdate (JRCaptureObject) returns YES, you can update the elements by calling updateOnCaptureForDelegate:context:.
