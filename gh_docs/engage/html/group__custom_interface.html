Detailed Description

Customize the user interface with your application's colors, images, native login, etc.

The Engage for iPhone SDK provides the ability to customize the look and feel of the user interface, as well as the ability to add your own native login experience, through the Custom Interface API. You can set many of the custom interface defaults through the setCustomInterfaceDefaults: (JREngage) method and override these defaults programmatically through any of the ...CustomInterfaceOverrides functions.

The SDK accepts a mutable dictionary of values, indexed by a pre-defined set of keys, and uses these to set the properties of the library's user interface. To configure the SDK programmatically (e.g., dynamically integrating your native login experience above or below the library's social logins), create an NSMutableDictionary object, indexed by a pre-defined set of keys and pass this to the library through the setCustomInterfaceDefaults: (JREngage) method.

You can also override these default customizations by launching the authentication and social sharing with a new dictionary of the pre-defined set of keys and values with the showAuthenticationDialogWithCustomInterfaceOverrides: (JREngage) or showSharingDialogWithActivity:withCustomInterfaceOverrides: (JREngage) methods. Any overlapping values passed into the show...Dialog methods will override the corresponding values passed into the setCustomInterfaceDefaults: (JREngage) method.

Define Documentation

#define kJRApplicationNavigationController

Key for your application's main UINavigationController object on which to push the sign-in and sharing dialogs

#define kJRCustomModalNavigationController

Key for a UINavigationController object that your application owns.

If you want to push your own views onto the library’s navigation controller, but your application does not have a navigation controller or your application is running on the iPad or if you want the library’s dialogs to present themselves using the UIModalTransitionStyleCoverVertical transition style, you can use the kJRCustomModalNavigationController key. This is also necessary if you want to tint the navigation bar’s color.

#define kJRNavigationControllerHidesCancelButton

Key for a boolean value represented by an NSObject to indicate if the dialog's "Cancel" button should be hidden.

If you want to hide the "Cancel" button on the Providers Table or Social Sharing screen, pass in an NSObject containing a positive integer or the strings @"YES" or @"TRUE". This will remove the "Cancel" button from the library's dialogs.

Warning:
Do not cancel social sharing or authentication by popping the navigation controller back to one of your views, as this could potentially leave the library in an unexpected state. If you wish to cancel sign in or social sharing, please use the cancelAuthentication (JREngage) or cancelSharing (JREngage) methods.
#define kJRAuthenticationBackgroundColor

Key for a UIColor object to set as the background color of the Providers Table and the User Landing screen.

#define kJRSocialSharingBackgroundColor

Key for a UIColor object to set as the background color of the Social Sharing screen.

#define kJRAuthenticationBackgroundImageView

Key for the UIImageView containing the image to be set as the background of the Providers Table and the User Landing screen.

#define kJRSocialSharingBackgroundImageView

Key for the UIImageView containing the image to be set as the background of the Social Sharing screen.

#define kJRProviderTableTitleView

Key for the UIView object to be set as the title view of the Providers Table.

Note:
If this value is set, it will override any string value set for define kJRProviderTableTitleString, although the define kJRProviderTableTitleString value will be used as the text on the "Back" button.
#define kJRSocialSharingTitleView

Key for the UIView object to be set as the title view of the Social Sharing screen.

Note:
If this value is set, it will override any string value set for define kJRSocialSharingTitleString, although the define kJRSocialSharingTitleString value will be used as the text on the "Back" button.
#define kJRProviderTableTitleString

Key for the NSString title to be set as the title of the Providers Table.

Note:
If a UIView* is set for the define kJRProviderTableTitleView key, this string will not appear as the title on the navigation bar. It will only be used as the text on the "Back" button.
#define kJRSocialSharingTitleString

Key for the NSString title to be set as the title of the Social Sharing screen.

Note:
If a UIView* is set for the define kJRSocialSharingTitleView key, this string will not appear as the title on the navigation bar. It will only be used as the text on the "Back" button.
#define kJRProviderTableHeaderView

Key for the UIView object to be set as the header view of the Providers Table.

#define kJRProviderTableFooterView

Key for the UIView object to be set as the footer view of the Providers Table.

#define kJRProviderTableSectionHeaderView

Key for the UIView object to be set as the view of the providers section header in the Providers Table.

Note:
Setting this value overrides any string set as the define kJRProviderTableSectionHeaderTitleString.
#define kJRProviderTableSectionFooterView

Key for the UIView object to be set as the view of the providers section footer in the Providers Table.

Note:
Setting this value overrides any string set as the define kJRProviderTableSectionFooterTitleString.
#define kJRProviderTableSectionHeaderTitleString

Key for the NSString to be set as the title of the providers section header in the Providers Table.

Note:
If a UIView* is set for the define kJRProviderTableSectionHeaderView key, this string will not be used.
#define kJRProviderTableSectionFooterTitleString

Key for the NSString to be set as the title of the providers section footer in the Providers Table.

Note:
If a UIView* is set for the define kJRProviderTableSectionFooterView key, this string will not be used.
#define kJRPopoverPresentationFrameValue

Key specifying the NSValue of a CGRect from which the authentication and sharing dialogs should originate if using a modal popover view on the iPad.

Example:
 CGRect rect = CGRectMake(x,y,w,h);
 NSValue *rectValue = [NSValue valueWithCGRect:rect];
#define kJRPopoverPresentationBarButtonItem

Key for the UIBarButtonItem object from which the authentication and sharing dialogs should originate if using a modal popover view on the iPad.

#define kJRPopoverPresentationArrowDirection

Key for an NSNumber object representing the UIPopoverArrowDirection enumeration when presenting the dialog from a UIPopoverController.

The default is UIPopoverArrowDirectionAny.

Example:
 [NSNumber numberWithInt:UIPopoverArrowDirectionDown];
#define kJRRemoveProvidersFromAuthentication

Key for an NSArray object containing a list of /e NSString provider names that you would like to exclude from table of providers when you launch the sign-in dialog

For a list of possible strings, please see the List of Providers

#define kJRCaptureConventionalSigninTitleString

Key for the NSString title to be set as the title of the Native Sign-in view.

Note:
If a UIView* is set for the define kJRCaptureNativeSigninTitleView key, this string will not appear as the title above the table section with the Capture Native Sign-in view, but the customized title view will be used instead. If you want to customize the title that will appear over the table section displaying the social providers, you can use the keys define kJRProviderTableSectionHeaderTitleString or define kJRProviderTableSectionHeaderView
#define kJRCaptureConventionalSigninTitleView

Key for the UIView object to be set as the title view of the Native Sign-in view.

Note:
If this value is set, it will override any string value set for define kJRCaptureNativeSigninTitleString key. If you want to customize the title view that will appear over the table section displaying the social providers, you can use the keys define kJRProviderTableSectionHeaderTitleString or define kJRProviderTableSectionHeaderView
