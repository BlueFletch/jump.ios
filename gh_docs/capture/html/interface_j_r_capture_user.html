- (BOOL) needsUpdate  

Use this method to determine if the object or element needs to be updated remotely. That is, if there are local changes to any of the object/elements's properties or sub-objects, then this object will need to be updated on Capture. You can update an object on Capture by using the method updateOnCaptureForDelegate:context:.

Returns:
YES if this object or any of it's sub-objects have any properties that have changed locally. This does not include properties that are arrays, if any, or the elements contained within the arrays. NO if no non-array properties or sub-objects have changed locally.
Note:
This method recursively checks all of the sub-objects of JRCaptureUser:
  • JRCaptureUser.primaryAddress
If any of these objects are new, or if they need to be updated, this method returns YES.
Warning:
This method recursively checks all of the sub-objects of JRCaptureUser but does not check any of the arrays of the JRCaptureUser or the arrays' elements:
  • JRCaptureUser.photos, JRPhotosElement
  • JRCaptureUser.statuses, JRStatusesElement
  • JRCaptureUser.profiles, JRProfilesElement
If you have added or removed any elements from the arrays, you must call the following methods to update the array on Capture: replacePhotosArrayOnCaptureForDelegate:context:, replaceStatusesArrayOnCaptureForDelegate:context:, replaceProfilesArrayOnCaptureForDelegate:context:
Otherwise, if the array elements' JRCaptureObject.canBeUpdatedOnCapture and needsUpdate (JRCaptureObject) returns YES, you can update the elements by calling updateOnCaptureForDelegate:context:.
